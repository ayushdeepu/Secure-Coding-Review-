Findings

General best practices

01

Never build queries with string formatting

Use parameterized queries or an ORM for every database call — no exceptions, even for "trusted" admin routes.

02

Treat all request data as hostile

Validate type, length, and format of every form field, query param, and header before it touches business logic.

03

Keep secrets out of source control

Load keys and credentials from environment variables or a secrets manager; rotate anything that was ever committed.

04

Hash passwords with a slow, salted algorithm

Use bcrypt, scrypt, or Argon2 — never MD5 or SHA-family alone — via a maintained library such as Werkzeug's security helpers.

05

Let the template engine escape output

Use Jinja2's autoescaping (render_template, not render_template_string with manual string concatenation) to prevent XSS.

06

Run static analysis on every commit

Wire Bandit (or similar) into CI so a vulnerable pattern fails the build before it reaches a reviewer.
